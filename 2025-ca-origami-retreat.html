Q: How to get to a state of complete bliss and delight and peace while working with (big) data?

Make re-use convenient

What makes data re-use INconvenient?

  • Data bugs. Possible origins:
    • Human entry
      Solution - minimize:
      • pre-scribe (standardize): ReproIn, BIDS, Nipoppy, ...
      • automate: HeuDiConv, ReproStim, ReproEvents,...
    • Software (human code)
      Solution avoid creating new, re-use (needs standardized interfaces),
      automate and standardize! testing.
    • Hardware (made by humans)
      Solution:
      • automate detection - write software to auto-detect (QA/QC. Join INCF niQC)
      • engage with manufacturers and their standards (DICOM):
        WiP to join IHE SHARAZONE.

What makes data re-use INconvenient?

  • Ad-hoc data access
    Solution:
    • standardize (HTTP 401/403, OAuth, etc),
    • abstract behind common interface (fsspec, DataLad)
    • think about both humans (UI) and bots (API)
  • Crafty data format/organization
    Solution: standardize, simplify and automate organization (ReproNim/ReproIn )
  • Opinionated software
    Solution: avoid creating new; IO and interfaces ideally should
    use standards (e.g., Boutiques, BIDS/BIDS-Apps, BrainLife ABCD)

What allows to make data re-use convenient?

  • Standards
    • The language(s) for humans and/or computers
    • Multilingual support
  • Validation
    • To make sure that you & others can understand the data

What allows to make data re-use convenient?

Standards: BIDS

  • UNIX Philosophy #1: "Make each program do one thing well" applies
  • Reuse/improve/adapt/convert/... instead of re-implement!
  • Accompany with tools to manipulate
    • desperately still needed for BIDS
  • Accompany with tools to validate
  • Accompany with tools to operate on (BIDS-Apps)
